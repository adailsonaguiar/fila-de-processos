Algoritmo FCFS ( First-Come-First-Serced)

No algoritmo FCFS (First Come First Served), o processo que chega primeiro é enviado primeiro para execução pela CPU quando a CPU está livre.
Desta forma há uma fila ordenada por Timestamp, assim, quando um novo processo requer memória, aquele que tiver o Timestamp menor( ou seja está a mais tempo na Mémoria) é escolhida como vítima.

Algoritmo de Histórico de Referência

No algoritmo de Histórico de Referência verifica-se se houve solicitação àquela página nos últimos momentos representados pelo histórico de bits de referência, assim, antes de realizar a substituição da página.
Toda a lista de páginas é percorridas de forma a se verificar se a página foi acessada no último momento, caso haja empate, usa-se o HISTÓRICO para retroceder no tempo de interações a fim de determinar qual página foi menos acessada.
Desta forma as páginas que tem o bit de referência 1 (indica que foram acessadas recentemente) já o bit de referência 0 (não foi acessada recentemente), dentre as páginas não foram acessadas caso haja mais de uma, verifica-se o acesso entre elas no tempo i-1, e assim persiste até que se encontre uma página menos acessada em relação a outra, em caso de empate, aquela que tiver o TimeStamp menor (aquela que está a mais tempo na memória) será a página vítima escolhida .

Algoritmo de Segunda Chance

No algoritmo de Segunda Chance verifica-se o bit de referência antes de realizar a substituição da página.
Todas as páginas são percorridas de forma circular em uma fila ordenada por Timestamp, de forma que as páginas que tem o bit de referência 1 (indica que foram acessadas recentemente) recebem uma segunda chance e vão para o final da fila (timestamp atualizado e bit de referência alterado para 0). A primeira página que tem o bit de referência 0 (não foi acessada recentemente) é escolhida como vítima.

Memória lógica

Quando um programa é executado, um processo é criado e seu espaço de endereçamento é alocado na memória lógica. Cada processo possui uma memória lógica contígua, dividida em páginas lógicas de mesmo tamanho.

O conteúdo referente às páginas lógicas são carregados em páginas físicas (ou quadros), não necessariamente contíguos.

Uma tabela de páginas associa as páginas lógicas com as páginas físicas.

Memória Física

A memória RAM (Random Access Memory) é um componente físico acoplado à placa mãe. Nela, são armazenados os dados dos processos.

Na paginação pode demanda, a memória RAM é dividida em páginas físicas, representada aqui por uma tabela contendo 8 linhas, em que cada linha representa uma página física. O conteúdo referente às páginas lógicas são carregados nas páginas físicas.

Considerando que espaço físico da memória é limitado, apenas páginas lógicas utilizadas são carregadas para a memória física, assim, o disco (HD) é utilizado como um mecanismo de memória virtual para armazenar todo o conteúdo do processo.

Disco

Mecanismos modernos de paginação como a paginação por demanda, permitem que quando um processo for criado, apenas as páginas que serão utilizadas no momento da execução sejam carregadas para a memória principal.
As demais páginas, assim como todas as páginas do processo (inclusive as páginas na RAM), ficam armazenadas no disco. Caso uma página que não esteja na RAM seja referenciada, uma cópia dela é feita para a memória principal e a tabela de páginas é atualizada para que seu acesso seja disponibilizado.
